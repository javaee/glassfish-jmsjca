What is JMSJCA?

JMSJCA is J2EE 1.4 Resource Adapter for JMS: it provides the interfacing between a J2EE or EE application server and a JMS Server. JMSJCA can also be used as a standalone library outside of an application server.

JMSJCA ships with CAPS 5.1.x and 6, with GlassFish ESB (both as a JCA deployed in GlassFish, and as an integral part of the JMS BC), and JMS Grid. JMSJCA is available as a single resource adapter (rasunone.rar, rastcms.rar, etc) for each supported JMS Provider, or as a single unified resource adapter (sun-jms-adapter.rar) that works with all supported JMS Providers.

Features

  • Spec compliant: tested with the Sun J2EE TCK (Sun SeeBeyond Integration Server / STCMS, Sun Java System Application Server 8.x / JMS Grid)
  • Advanced connection error detection and handling
  • Asynchronous deployment
  • Feature rich Dead Letter Channel (aka deadletter queue)
  • Batch Processing
  • JMX MBean support
  • Multiple concurrency modes: serial, sync, connection consumer
  • Advanced transaction support: XA emulation
  • Supports multiple JMS Providers (click to expand)
    • Sun Java System Message Queue, also known as Open MQ
    • STCMS, also known as Sun / SeeBeyond Intelligent Queue Manager, part of Java CAPS
    • STCMS 4.5.3
    • Websphere MQ Series
    • Wave, also known as JMS Grid, shipping as part of Java CAPS
    • WebLogic JMS
    • JBoss JMS
    • Any JMS Provider of which the connection factories can be bound in JNDI
  • Built-in workarounds for problems with these JMS Providers
  • Works with multiple J2EE / Java EE application servers (click to expand)

    The following J2EE servers have been tested (A=full test suite, B=reduced test suite):
    • Sun Java System Application Server 8.x (A)
    • Glassfish (A)
    • Sun SeeBeyond Integration Server, shipping as part of Java CAPS (A)
    • Weblogic 9.x (A)
    • JBoss 4.x (B)
    • WebSphere 6 (B)
  • Can be used outside of an application server in the form of a jar (click to expand)
    • Built-in JCA container:
    • Work manager with threadpool (based on Java SE 1.5 concurrent library)
    • ConnectionManager
      • Connection pooling
      • Supports XA
      • Automatically enlists and delists connections in transactions
      • Takes care of the problem of keeping connections open after an application calls close() in a transaction

System requirements and dependencies

  • JDK 5 or higher
  • JMS 1.0.2 or JMS 1.1 JMS Provider
  • J2EE 1.4 application server, or use the built-in JCA Container

Roadmap

JMSJCA is to merge with the JMSRA and GenericJMSRA and will ship as the one and only JMS connector with GlassFish v3. This consolidation effort was started in H2 of 2008 and will finish in H2 of 2009.

Developing JMSJCA

Source code access

Browse the Maven repository for zip files of the source. Sources can also be checked out from CVS.

Building JMSJCA

Building requires Ant 1.6 or higher. All external files (e.g. javaee.jar) are automatically downloaded. Type ant -projecthelp for an overview of all available targets.

Testing JMSJCA

Running the JMSJCA test suite requires the jars of the JMS Providers and EE application servers being tested. See ant -projecthelp for more information.
