What is JMSJCA?

JMSJCA is a a very feature-rich library that abstracts differences between JMS servers and provides a single interface to these JMS servers. It currently ships as part of  Java CAPS, and JMS Grid, in the form of a resource adapter, and as part of Open-ESB in the form of a Binding Component (JMSBC).


This project is part of the open-jbi-components project.

Some of the more advanced features include:
- deadletter qeueue
- advanced connection failure detection and handling
- asynchronous deployment
- JMX MBean support
- concurrency modes
- advanced transaction support
see the JMSJCA release notes for a complete list of features.

JMSJCA supports the following JMS servers:
- Sun Java System Message Queue, also known as Open MQ
- Generic JNDI
- STCMS, also known as Sun / SeeBeyond Intelligent Queue Manager, part of Java CAPS
- STCMS 4.5.3
- Websphere MQ Series
- Wave, also known as JMS Grid, shipping as part of Java CAPS
- WebLogic
- JBoss

The following application servers are supported:
- Sun Java System Application Server 8.x
- Glassfish
- Integration Server, shipping as part of Java CAPS
- Weblogic 9.x
- JBoxx 4.x
- WebSphere 6
